サイトの概要・使用技術
サイト名：Mt.Practice (マウントプラクティス)
サイト概要：同じ教材を持っている人同士をつなげる教材に特化したQ&Aサイト
サイトURL： https://mt-practice.com
使用言語：Java
フレームワーク：Spring Boot, mybatis
フロントエンド： HTML/CSS, JavaScript, jQuery, Bootstrap, thymeleaf
バックエンド：AWS(EC2, RDS, S3), MySQL(ver8.0.28), Apache HTTP Server, Apache Tomcat
環境・使用ツール：Windows(local), Linux2(remote/Ec2), Eclipse, GitHub
作成期間：2年 (構想段階含む)
使用技術選定理由：自分の持ち得る技術を最大限に発揮しようと考えたため。
※実務未経験のため当サイトの使用技術が保持スキルと同等となりますが、オンライン教材を使用したKotlin、PHPの基本文法の学習経験もございます。
なぜこのサイトを企画したのか
　私自身の職業訓練校(Javaプログラミング)での学習過程で、授業についていけずに悩んだ挫折経験が元になっています。 分からない時にはQ&AサイトやGoogle検索がとても役に立ちますが、欲しいヒントを見つけ出し、自分の状況に置き換える作業にいつも多くの労力や時間がかかってしまい、目標の量がこなせなくなるという悩みを持っていました。
　そんな時にふと、「教材のタイトルごとに分けられたQ&Aサイトがあったら助かるのにな」と思ったことをきっかけに、自分のためだけでなく同じ悩みを抱える学習者のためにも作る意義を感じたためです。
サイトの目的
　意欲的に学習に取り組んでいても、「目標の課題の量がこなせない」「量はこなせていても理解が追い付かない」という悩みを抱える学習者をサポートすること。
このサイトが解決できること・期待できる効果
＜悩みを抱えるユーザー(メインターゲット)＞
・より多くの仲間に向けて質問ができ、自分の理解に必要な回答が得やすくなる。
→質問をできる仲間が自分の所属組織や所属範囲の垣根を超えて広がるため。
→各組織ごとや教材ごとのQ&Aシステムでは管理者対ユーザーの縦のやり取りが主体だが、 第三機関が作るQ&Aシステムであることによって気兼ねなくユーザー同士の横のやり取りができるようになるため。
→回答をもらえる量やスピードだけでなく、自分と同じレベルの人からのアドバイスも期待できることで、分かりやすさという回答の質にもつながるため。

・必要なヒントを検索する時に、絞り込み作業にかかる労力や時間を大幅に削減できる。
→同じ教材を使っている者同士であれば、同じ疑問を持ち、同じヒントを探している確率も高く、 個々に同じヒントを探す手間を1つのサイトに集約する事で探す手間を減らせるため。

・従来のQ&Aサイトより質問時の状況説明を簡単に行うことができる。
→同じ教材を使っている人同士がつながることで問題文等の説明の省略ができるため。

・同じ教材を使っている人同士で、学習方法等の相談もする事ができる。
→質問投稿時のページ入力欄で、ページ番号「0」を特定のページ以外の相談用にするという共通ルールにより対応できるようにしたため。

・自信を失うことに繋がるネガティブ要因を抑制し、安心して質問できる。
→やりとりの際の言葉遣いや「否定をしない」というサイト独自のルールを設けているため。
※イメージ図 (全国で市販されている教材の場合)

＜悩みを抱えていないユーザー(サブターゲット)＞
・教える側も教える事で復習となり理解がより深まる。
・情報が集約化されているため、スムーズに参考情報を集められるツールとして使える。
・サイト内で教える程自分の勉強のしやすい環境を生む。
→サイト利用者の増加につながり、今後別の教材で学習する時の「自分が質問できる相手」や「参照できるＱ＆Ａ履歴数」の増加に繋がるため。
＜出版社・版元(サブターゲット)＞
・サイトのQ&A履歴やデータからユーザーの生の声に寄り添った改版が行え、より良い教材作りに役立てられる。
・エンドユーザーとのコネクションツールとしても利用できる。
・教材に「安心」「便利」といった付加価値を付けることができる。
・販促物として売上アップに繋げられる。
・教育業界全体で取り組めば、教育市場規模の拡大が見込める。
→「できる、分かる人」が増えれば学習に積極的になる人も増え、1人あたりの学習機会の増加につながるため。

工夫した点
＜UIデザイン＞
　スマートフォン環境による学習環境が多い事を想定して、ユーザー目線で実機で何度もチェックを行いながらレスポンシブ対応を行いました。
　またQ&A一覧では質問者が回答をもらいやすくなる工夫として、回答の有無や解決の有無を一目で判別できるような配色デザインにしました。
　サイトのロゴやタイトルは「同じ教材の山を助け合って登る」というコンセプトの価値提案を意識してデザインしました。

＜サイト独自のアイデア＞
　「セルフ回答ボタン」を作ることで、分からない学習者が参照できるヒントを1つでも多くするための工夫を行いました。
　学習方法の悩み相談等にも対応し、質問投稿時のページ入力欄に0を入れるだけで、他の質問と区別しながらも簡単に行える仕組みを作りました。
　また、悩みを抱える学習者の多くは自信を欠いている状態のため、やりとり時のルールを制定することで安心して質問できる環境作りの工夫も行いました。
苦労した点
　教材を手本に作り上げたサーバーがウィルスの感染を受けてしまい、作り直しになってしまったことです。 基礎知識や予備知識があれば対処出来ていたことでしたが、この経験の場面こそ自ら作るWebサイトが役に立てる場面であると、制作意義を再認識することができました。
今後の課題
　現段階ではパブリックな市販の教材向けの仕様になっておりますが、プライベートな社内研修用教材や学習塾専用教材、 他にもサイトやブログを教材に見立てた需要も考えられます。こういった様々な需要に対応できるように。予めシステム設計を準備しておくことが今後の課題であると考えております。
目指す状態
このサイトが悩みを抱える学習者にとって心強いサポートサイトになっており、
世の中に「同じ教材を持っている人同士がネット上でつながり、助け合う」というカルチャーが浸透している状態。
